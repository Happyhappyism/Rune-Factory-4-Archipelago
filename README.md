# Setup Guide
## Necessary Software: 
[Archipelago Latest Release](https://github.com/ArchipelagoMW/Archipelago/releases/latest)
[quick bms](https://aluigi.altervista.org/quickbms.htm)
RF4S APworld `rune4.apworld`

## Setup steps:
- Create a folder named "Archipelago" in your Rune Factory 4 Special install directory.
Extract Quickbms somewhere on your machine where it won't get moved. Personally, I put it into my install directory as well.
It should look something like the picture below.
<img width="1561" height="679" alt="foldersetup" src="https://github.com/user-attachments/assets/1c586eb5-5ee8-4214-9828-448fd5ca5ae7" />

- Install the RF4S AP world (This can be done by simply double clicking the apworld wherever you downloaded it to. If it asks for a program to open it with, choose the Archipelago launcher.)
- Open your AP launcher and select "Generate Template files", which will make your RF4S Template and open your template folder.
- Set up your yaml, and when done, place it into your "players" folder, then generate your game (or send the yaml to the person that will be hosting if you'll be playing a multiworld.)
- Generate your game.
- In the output zip from generation, there will be a save marked with your chosen player name. Place that save into the Archipelago folder we made in your install directory. If you have Trupin hints enabled, also place your hints.json file in the same Rune Factory 4 Special\Archipelago folder as well.
- Run the RF4S Client from the AP launcher. This will prompt you for the paths to your install folder, the BMS folder (which if you followed this guide is probably inside of your install folder), and your RF4S saves folder, which can usually be found at C:\Users(your username here)\AppData\Roaming\Rune Factory 4 Special
- Once given the right paths, the client will automatically generate your save, back up the save it replaces, and open RF4S. From there, you need only connect to your server and play the game.
